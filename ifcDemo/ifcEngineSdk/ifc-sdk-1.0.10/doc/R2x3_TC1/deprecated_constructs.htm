Deprecated IFC Constructs

The purpose of this section is to tabulate in one place all the deprecated constructs identified in this specification and to clarify the meaning of deprecation in the context of the IFC.

This specification, Technical Corrigendum 1 of Edition 3 of IFC2x, contains all of the IFC2x Platform on which it is based. The IFC2x Platform was released with the intent to establish a set of schemas which would remain unchanged through subsequent editions except for extensions in capability. However, some of the information structures defined in the IFC2x Platform, or some specific uses of them, have been found deficient or have been superseded by new capabilities introduced in subsequent editions. These constructs are now deprecated and have been identified as such where they are defined in the specification.

Deprecation of an information structure or some specific use of it means that it shall not be used in originating applications that are conformant to this specification. Hence instances of these deprecated constructs shall not occur in an exchange file or repository that is conformant to this specification. Receiving applications shall not be required to process instances of the deprecated concepts into meaningful internal representations. Application developers are free to choose how to deal with such instances should they occur. Commercial software tools exist that can be used to verify exchange files and repositories for conformance to these and other requirements based on appropriate rulesets. A list of IFC software tools is maintained online and can be accessed through the “Software” tab at http://www.iai-international.org.
